Practical Hands-On Projects

Reading time: calculating... | Portfolio Builders for CS Students

1. Software Project: Carbon-Aware Web App

The Goal: Build a tool that modifies its behavior based on the live carbon intensity of the local power grid.

How to build it: Register for a free API key from WattTime or Electricity Maps. These APIs provide real-time, locational Marginal Operating Emissions Rate (MOER) data. Build a simple Python/Flask or Node.js web dashboard that checks the API.

Extension: Write a Python script that only allows a long-running rendering task (or a local ML model training run) to execute if the local grid's renewable energy mix is above 50%.

2. Hardware Project: DIY PM2.5 Air Quality Node

The Goal: Learn embedded systems by building a cheap IoT air quality monitor that feeds data to a global public map.

How to build it: Order an ESP32 microcontroller (with built-in WiFi), a Plantower PMS5003 laser dust sensor, and a BME280 humidity/temperature sensor (total cost ~$35). Wire them together using jumper cables on a breadboard. Write a C++ script using the Arduino IDE to read the serial data from the PMS5003.

Extension: Use the MQTT protocol to send your sensor's live data to a cloud database, or push the data to a public citizen-science API like Sensor.Community (formerly Luftdaten) or PurpleAir.

3. Data Project: Urban Tree Canopy Mapping (GIS)

The Goal: Analyze spatial inequality in urban heat islands using Geographic Information Systems (GIS).

The OpenStreetMap Workflow

Download QGIS (a free, open-source geospatial software suite). Download open-source vector shapefiles of your city's street trees from their Open Data portal, and download the city's census income tracts.

Write a Python script using the Geopandas library to perform a spatial join, calculating the density of tree canopy cover per capita overlaid against median household income. Create a heatmap visualization proving whether wealthier neighborhoods have disproportionately more shade (a common reality due to historic redlining).

4. Contributing to Open Source Climate Software

If you don't want to start from scratch, massive open-source communities already exist.

  • Open Climate Fix: An open-source lab attempting to reduce emissions by gigatons by building open ML models for forecasting solar clouds. They actively need Python/PyTorch contributors.
  • ClimateTriage: A directory specifically built to match developers with open-source climate projects that have open "Good First Issues" on GitHub.
  • OpenStreetMap (OSM): The "Wikipedia of Maps." You can create an account today and start tracing the outlines of solar panels on satellite imagery or logging the coordinates of public EV chargers in your town, adding to the free global database used by researchers worldwide.

Key Takeaways

  • You can practice Green Software Engineering today by hooking into free grid-carbon APIs like WattTime.
  • Building embedded hardware (like an ESP32 air sensor) connects abstract code to physical urban realities.
  • Geospatial tools like QGIS and Geopandas are essential for analyzing systemic climate inequalities.
  • Open-source communities desperately need traditional software engineering help to scale climate models.
